# Reconstructive Burden

Reconstructive burden measures the effort required to rebuild a behavioral model from fragmented documentation.

It includes:

- cross-navigation depth
- terminology instability
- hidden dependencies
- async opacity
- missing states
- divergent evidence classes
- conflict density

## Example

A payment approval workflow has high reconstructive burden if understanding it requires:

1. a UI guide,
2. an API reference,
3. a FAQ,
4. a support article,
5. and undocumented assumptions about role equivalence.
