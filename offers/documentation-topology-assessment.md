# Documentation Topology Assessment

## Objective

Identify how documentation surfaces interact, diverge, or fragment behavioral understanding.

## Surfaces analyzed

- API reference
- guides
- UI documentation
- support articles
- FAQs
- release notes
- examples
- error messages

## Output

A topology map showing:

- stable behavior zones
- fragile behavior zones
- unsupported assumptions
- cross-page dependencies
- scope leakage
- terminology drift
