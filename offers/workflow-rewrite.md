# Workflow Documentation Rewrite

## Purpose

Rewrite a critical product or API workflow so users can understand not only the steps, but the behavior behind the steps.

## Best fit

This offer is useful when:

- users can follow steps but cannot predict outcomes
- workflows depend on roles, states, or conditions
- documentation is split across several pages
- onboarding requires support intervention
- API integration requires trial and error

## Scope

Rewrite one focused workflow.

Examples:

- payment approval workflow
- API authentication and retry workflow
- user role configuration
- webhook integration
- data sync setup
- migration flow

## Method

1. Audit current workflow documentation
2. Identify missing relationships
3. Create behavior model
4. Restructure page sequence
5. Rewrite selected sections
6. Add examples, edge cases, and links
7. Deliver before/after comparison

## Deliverables

- rewritten workflow structure
- behavior matrix
- before/after documentation example
- improved page outline
- recommendations for linked pages
- optional Markdown-ready content

## Duration

Typical duration:

```text
2 to 4 weeks
```

## Not included

- rewriting the entire documentation site
- product UI copy
- engineering implementation
- analytics instrumentation

## Value

The rewrite makes a workflow executable and predictable.

It reduces onboarding friction and support dependency.
