# Articles

These articles give the conceptual and methodological frame behind the portfolio case studies.

They are not archived LinkedIn posts. They are portfolio pages that explain how documentation can make product behavior, conditions, dependencies, exceptions, and AI-retrievable knowledge explicit.

## Portfolio function

These articles support the case studies by defining the recurring problem: users and teams reconstruct system behavior from fragments.

They also define the documentation response: make product behavior explicit through stable terms, visible dependencies, mapped conditions, documented exceptions, and recoverable workflows.
